# 稀疏观测策略包 - 输出文件说明

本目录包含稀疏观测策略包的输出结果，用于分析基站能耗数据的稀疏性特征。

## 输出文件列表

### 1. `ec_coverage_by_bs.csv`

**描述**：每个基站的EC数据覆盖情况分析

**字段说明**：

- `BS`：基站ID
- `n_obs`：观测数量（数据点个数）
- `time_min`：最早观测时间
- `time_max`：最晚观测时间
- `gap_median_h`：时间间隔的中位数（小时）
- `gap_p90_h`：时间间隔的90百分位数（小时）
- `gap_max_h`：最大时间间隔（小时）
- `prop_step_eq_1h`：时间间隔等于1小时的比例（正则步长比例）
- `ideal_hour_slots`：理想情况下的小时数（从time_min到time_max的总小时数）
- `obs_to_ideal_hour_ratio`：观测数与理想小时数的比例（观测占比）

**用途**：评估每个基站的数据完整性和时间分布特征

### 2. `merged_gap_by_bs.csv`

**描述**：合并后数据的时间间隔分析

**字段说明**：与`ec_coverage_by_bs.csv`相同，但基于合并后的数据

**用途**：评估合并后数据的时间连续性，识别数据稀疏性问题

### 3. `bs_below_hour_threshold.csv`

**描述**：观测数量低于阈值的基站列表

**字段说明**：

- `BS`：基站ID
- `n_obs`：观测数量

**用途**：识别数据量不足的基站，这些基站可能需要特殊处理或剔除

### 4. `fig_sparse_energy_B_0_2023-01-01_2023-01-08.png`

**描述**：B_0基站的能耗曲线示例图

**特点**：

- 显示了B_0基站在2023年1月1日至1月8日的能耗曲线
- 不插补数据，在间隔>1小时处显示断线
- 直观展示数据稀疏性

**用途**：可视化基站能耗数据的时间分布和稀疏性特征

### 5. `global_summary.json`

**描述**：全局摘要统计信息

**内容**：

- 总站数
- 总观测数
- 全局时间间隔分布
- 其他全局统计指标

**用途**：提供整体数据质量评估，便于论文附录或报告使用

### 6. `run_summary.csv`

**描述**：运行摘要信息

**内容**：

- 运行时间
- 处理的数据量
- 主要处理步骤的统计信息

**用途**：记录运行过程和结果，便于后续分析和复现

## 数据稀疏性评估指标说明

### 关键指标解释

1. **正则步长比例** (`prop_step_eq_1h`)
  - 取值范围：0-1
  - 接近1表示数据时间间隔大部分为1小时，数据较为连续
  - 接近0表示数据时间间隔不规则，数据较为稀疏
2. **观测占比** (`obs_to_ideal_hour_ratio`)
  - 取值范围：0-1
  - 接近1表示观测覆盖了大部分理想时间槽
  - 接近0表示观测覆盖度低，数据稀疏
3. **时间间隔分布** (`gap_median_h`, `gap_p90_h`, `gap_max_h`)
  - 中位数和90百分位数反映了典型的时间间隔
  - 最大值反映了数据中的最大缺失窗口

### 数据质量评估标准

- **优质数据**：`prop_step_eq_1h > 0.8` 且 `obs_to_ideal_hour_ratio > 0.8`
- **中等数据**：`0.5 < prop_step_eq_1h <= 0.8` 或 `0.5 < obs_to_ideal_hour_ratio <= 0.8`
- **稀疏数据**：`prop_step_eq_1h <= 0.5` 且 `obs_to_ideal_hour_ratio <= 0.5`

## 使用建议

1. **数据筛选**：根据`bs_below_hour_threshold.csv`剔除观测数过少的基站
2. **模型训练**：
  - 对于优质数据：可以直接使用常规时间序列模型
  - 对于中等数据：建议使用能处理缺失值的模型或采用时间对齐策略
  - 对于稀疏数据：建议采用稀疏观测策略，仅在真实观测点上训练
3. **特征工程**：
  - 考虑使用`gap_hours`作为特征，捕获时间间隔信息
  - 对于滞后特征，使用显式时间差过滤确保语义正确性
4. **可视化**：使用稀疏断线图直观展示数据质量，避免插补导致的误导

## 注意事项

- 本分析基于原始观测数据，未对Energy进行任何插补
- 时间间隔分析基于排序后的时间序列，反映了实际观测的时间分布
- 观测占比仅作参考，不代表数据质量的绝对标准
- 对于稀疏数据，建议在模型训练中使用mask特征标记真实观测点
