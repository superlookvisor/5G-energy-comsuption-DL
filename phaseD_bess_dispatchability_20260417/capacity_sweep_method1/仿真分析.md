# V. 仿真与结果分析

本章基于第 III 节提出的可靠性-可调度性联合评估框架以及第 IV 节给出的统一容量倍率扫描方法（下文记为方法 I），在 53 座 5G 基站、共 266 条测试集停电决策样本上开展数值实验，定量对比四种代表性后备能量需求刻画策略在不同 BESS 装机容量下的通信可靠性、可调度容量释放能力与净调度收益，并由此确定全网统一装机的最小可行容量倍率 $\chi^\star$。

## A. 仿真设置

修复时间服从参数为 $\mu=0.7$ 的指数分布，目标通信可靠性设为 $R^{\min}=0.99$，由此导出连续意义下的最小支撑时长 $T^{\mathrm{rel}}=-\ln(1-R^{\min})/\mu\approx 6.58$ h，离散化为 $T^{\mathrm{rel}}_{\mathrm{disc}}=7$ h。累计能耗不确定性的风险水平取 $\varepsilon_E=0.10$，用于在校准集上估计分位数 $q_{T^{\mathrm{rel}}}$；停电到达率 $\lambda_{\mathrm{out}}=0.01$ h$^{-1}$，中断惩罚系数 $\pi_{\mathrm{int}}=10$、未服务业务惩罚 $\pi_{\mathrm{ut}}=1$、调度收益单价 $\pi_d=1$。SOC 上下限分别为 1.0 和 0.0。每个基站 BESS 装机容量按 $C_b(\chi)=\chi\,p_b^{\mathrm{base}}$ 取值，其中 $p_b^{\mathrm{base}}$ 为 Phase A 估计得到的静态基础功耗。统一容量倍率扫描集为 $\chi\in\{4,6,8,10,12,14,16,18,20,24\}$。

比较的四类策略及其后备能量需求 $E_b^{\mathrm{req}}$ 的构造方式为：（i）Oracle Reliability，$E_b^{\mathrm{req}}=R_b^{\mathrm{true}}(T^{\mathrm{rel}}_{\mathrm{disc}})$；（ii）Phase C 点预测，$E_b^{\mathrm{req}}=\hat R_b^{\mathrm{C}}(T^{\mathrm{rel}}_{\mathrm{disc}})$；（iii）可靠性感知 CQR（Reliability-Aware CQR），$E_b^{\mathrm{req}}=\hat R_b^{\mathrm{C}}(T^{\mathrm{rel}}_{\mathrm{disc}})+q_{T^{\mathrm{rel}}_{\mathrm{disc}}}$；（iv）Hourly Upper Reliability，$E_b^{\mathrm{req}}=\sum_{k=1}^{T^{\mathrm{rel}}_{\mathrm{disc}}}\hat U_{b,k}^{90}$；（v）Full BESS Backup，$E_b^{\mathrm{req}}=E_b^{\mathrm{usable}}$。以下将 Oracle 作为上界基准，Full BESS Backup 作为容量充裕性基线。

## B. 容量瓶颈区与可靠性违约

当 $\chi=4$ 时，五类策略的平均通信可靠性均为 0.890，可靠性违约率 $\phi_{\mathrm{rv}}=1.000$，平均自主运行时长仅 3.41 h，显著低于 $T^{\mathrm{rel}}_{\mathrm{disc}}=7$ h。此时除 Phase C 点预测有极少量样本（约 1.1 %）因请求量略低于可用容量而释放 0.045 单位调度容量外，其余策略的平均可调度容量均为 0。该现象表明：在低倍率区间，BESS 可用能量不足以覆盖 7 h 窗口内的累计能耗，系统运行在容量充裕性受限区，此时所有策略均退化为 Full BESS Backup，可靠性瓶颈由容量决定，而非由调度策略决定。这验证了文献 [X] 中关于“可靠性违约并非完全由调度策略造成”的判断。

当 $\chi$ 增加至 6 时，平均可靠性跃升至约 0.967，但违约率仍高达 0.883，说明目标阈值 0.99 远未被满足；同时 Phase C 点预测的违约率（0.977）明显高于其它四类策略（0.883），反映其对未来累计能耗的系统性低估。

## C. 可靠性与违约率随 $\chi$ 的演化

图 1 绘制了五类策略的平均通信可靠性随 $\chi$ 的变化曲线；图 2 给出对应的可靠性违约率。主要观察如下：

1）对于 Full BESS Backup 策略，可靠性从 $\chi=4$ 时的 0.890 单调上升至 $\chi=8$ 时的 0.989，并在 $\chi=14$ 处达到 0.9970，对应违约率由 1.000 降至 0。这反映了可靠性关于后备能量的单调递增与修复分布的饱和特性。

2）对于 Reliability-Aware CQR 策略，一旦容量进入充裕区（$\chi\ge 12$ 时容量不可行率 $\phi_{\mathrm{cif}}=0$），其平均可靠性稳定在 0.9921，违约率稳定在 0.1053。这一残余违约率与累计能耗分位数风险水平 $\varepsilon_E=0.10$ 高度一致，与保形预测的名义覆盖率 $1-\varepsilon_E$ 相吻合，表明在容量不再是瓶颈之后，CQR 的剩余失效主要来源于分位数未能覆盖的 10 % 条件偏差样本，属于预测不确定性本身的固有极限。

3）Hourly Upper Reliability 策略在 $\chi\ge 12$ 时平均可靠性稳定在 0.9934、违约率 0.0451，相较 CQR 略有提升，印证了逐小时上界相加相较于累计分位数的固有保守性。

4）Phase C 点预测的违约率在 $\chi\ge 12$ 之后长期停留在约 0.594，说明即使容量完全充裕，点预测本身的系统性低估也会使接近 60 % 的样本仍处于违约状态，显著高于目标容许违约率 $\bar\phi=0.01$。

5）Oracle 策略在 $\chi\ge 14$ 时违约率降为 0，但平均可靠性仍稳定在 0.9926 而非 1.0，这是由于修复时间 CDF 本身的饱和效应——即 $F_D(T^{\mathrm{rel}}_{\mathrm{disc}})$ 在 $T^{\mathrm{rel}}_{\mathrm{disc}}=7$ 的上界约为 $1-e^{-4.9}$。

## D. 可调度容量释放

图 3 展示了平均可调度容量 $\overline{C}_{\mathrm{disp}}$ 随 $\chi$ 的变化。在 $\chi=6$ 时，由于容量几乎全部作为后备保留，各策略的 $\overline{C}_{\mathrm{disp}}$ 均小于 2.44 单位；从 $\chi=8$ 起，CQR、Hourly Upper、Phase C 点预测和 Oracle 开始显著释放可调度容量。典型对比见表 I。

**表 I** $\chi=10$ 时各策略的关键指标对比

| 策略 | $\overline{R}$ | $\phi_{\mathrm{rv}}$ | $\overline{C}_{\mathrm{disp}}$ | $\phi_{\mathrm{cif}}$ | $\overline{V}_{\mathrm{net}}$ |
|---|---|---|---|---|---|
| Full BESS Backup | 0.9952 | 0.053 | 0.00 | 0.000 | $-0.07$ |
| Oracle Reliability | 0.9919 | 0.053 | 50.87 | 0.053 | 50.75 |
| Reliability-Aware CQR | 0.9920 | 0.113 | 36.85 | 0.064 | 36.74 |
| Hourly Upper Reliability | 0.9928 | 0.075 | 31.66 | 0.248 | 31.55 |
| Phase C Point Reliability | 0.9869 | 0.598 | 53.67 | 0.023 | 53.48 |

由表 I 可见，在同等容量 $\chi=10$ 下，CQR 相较 Hourly Upper 在可靠性略有牺牲（$0.9920$ vs $0.9928$）的代价下，可调度容量提高约 16.4 %（$36.85$ vs $31.66$），容量不可行率下降约 74 %（$0.064$ vs $0.248$）；而 Phase C 点预测尽管释放了最多的可调度容量，其 0.598 的违约率远超工程可接受阈值，不具备可部署性。

## E. 最小可行统一容量倍率 $\chi^\star$

以 $\bar\phi=0.01$ 为最大容许违约率，从扫描结果中筛选得到：对 Full BESS Backup 策略，$\chi^\star_{\mathrm{FB}}=14$，即全网统一配置 14 × p$_b^{\mathrm{base}}$ 基础功耗小时数的 BESS 即可在不参与调度的情况下满足 $R^{\min}=0.99$ 的通信可靠性目标。对 Reliability-Aware CQR 策略，在扫描上界 $\chi=24$ 之内违约率始终高于阈值（稳定在 0.1053），说明其剩余违约来源于预测不确定性本身，**单纯增加装机容量不能进一步降低该违约率**，必须通过加密校准集、提升点预测模型精度或收紧 $\varepsilon_E$ 才能突破此下限。

这一结论在工程上具有重要含义：$\chi^\star$ 并非对所有策略都存在；当策略依赖不完美的能耗预测时，$\chi^\star$ 的存在性与可及性由预测残差分布、而非由储能容量决定。该现象也解释了为何在 $\chi\in[12,24]$ 范围内 CQR 的违约率曲线完全平坦（图 2）。

## F. 可靠性-可调度性折中

图 4 以平均可靠性为横轴、平均可调度容量为纵轴，给出以 $\chi$ 标注的散点图。从中可归纳出三点：

1）在 $\chi\ge 12$ 后，各策略沿 $\chi$ 方向几乎呈垂直线移动，即**可靠性趋于饱和，可调度容量随容量线性增长**。这说明一旦越过 $\chi^\star_{\mathrm{FB}}$，增加装机容量不再改变可靠性水平，只改变可参与电网调度的余量。

2）在同一 $\chi$ 等值线上，四类策略呈现清晰的 Pareto 排序：Oracle（上界）$\succ$ CQR $\succ$ Hourly Upper $\succ$ Phase C 点预测（违约率不可接受）$\succ$ Full BESS Backup（调度为零）。CQR 在接近 Oracle 的可靠性水平下，提供了比 Hourly Upper 更大的可调度容量，构成**帕累托意义上的首选可部署策略**。

3）以 $\chi^\star_{\mathrm{FB}}=14$ 为参考点：CQR 的平均可调度容量为 119.11，相较 Hourly Upper 的 107.68 提升 10.6 %，相较 Full BESS Backup 的 0 释放了全部容量弹性，且平均可靠性仍保持在 0.9921 水平。

## G. 净调度收益分析

表 II 汇总了 $\chi^\star_{\mathrm{FB}}=14$ 处各策略的净调度收益 $\overline{V}_{\mathrm{net}}=\pi_d\,\overline{C}_{\mathrm{disp}}-\pi_{\mathrm{int}}\,\overline{D}_{\mathrm{int}}-\pi_{\mathrm{ut}}\,\overline{U}_{\mathrm{ut}}$ 的组成。由于在 $\chi=14$ 下平均期望中断时长 $\overline{D}_{\mathrm{int}}$ 已普遍低于 0.02 h、平均未服务业务量 $\overline{U}_{\mathrm{ut}}$ 低于 0.002，单位惩罚量级相较可调度容量显著偏小，导致 $\overline{V}_{\mathrm{net}}$ 主要由 $\overline{C}_{\mathrm{disp}}$ 决定；这也解释了 Phase C 点预测虽违约率极高（0.594），但其 $\overline{V}_{\mathrm{net}}$ 仍能达到 136.28 的现象。

**表 II** $\chi=14$ 时各策略的净调度收益分解

| 策略 | $\overline{C}_{\mathrm{disp}}$ | $\overline{D}_{\mathrm{int}}$ (h) | $\overline{U}_{\mathrm{ut}}$ | $\overline{V}_{\mathrm{net}}$ |
|---|---|---|---|---|
| Full BESS Backup | 0.00 | 0.00424 | $4.0\!\times\!10^{-5}$ | $-0.042$ |
| Oracle Reliability | 132.73 | 0.01064 | $7.8\!\times\!10^{-4}$ | 132.62 |
| Reliability-Aware CQR | 119.11 | 0.01135 | $9.8\!\times\!10^{-4}$ | 118.99 |
| Hourly Upper Reliability | 107.68 | 0.00942 | $5.6\!\times\!10^{-4}$ | 107.58 |
| Phase C Point Reliability | 136.47 | 0.01874 | $1.82\!\times\!10^{-3}$ | 136.28 |

这一结果提示：**在中低惩罚系数下，单独以净调度收益为目标会偏向激进的点预测策略**，进而接受极高的违约率；因此在容量规划中应同时约束 $\phi_{\mathrm{rv}}\le\bar\phi$，把可靠性违约作为硬约束而非软惩罚。若将 $\pi_{\mathrm{int}}$ 按通信业务的工程价值重新标定至显著更高量级，则 Phase C 点预测在 $\overline{V}_{\mathrm{net}}$ 维度的优势将被抹平，此时 CQR 将同时在可靠性和净收益维度占优；这一参数灵敏性与运营商实际 SLA 惩罚量级高度相关，是容量规划中的工程决策变量。

## H. 小结

综合 B–G 节的结果可得以下可直接用于工程决策的结论：(i) 默认容量（$\chi=8$）下全网存在显著可靠性缺口，需进行统一扩容；(ii) 以违约率 $\bar\phi=0.01$ 为阈值，Full BESS Backup 的最小可行统一倍率为 $\chi^\star_{\mathrm{FB}}=14$，此结果可作为 BESS 容量规划的**工程下界**；(iii) 在 $\chi\ge\chi^\star_{\mathrm{FB}}$ 区间，Reliability-Aware CQR 在几乎不损失可靠性（相对 Full BESS Backup 损失约 0.5 个百分点）的前提下，将原本保留作后备的容量中约 85 % 释放为可调度余量；(iv) Phase C 点预测尽管净收益最高，但其违约率远超工程可接受阈值，不建议用于需承担 SLA 责任的场景；(v) CQR 的残余违约率与 $\varepsilon_E$ 高度吻合，表明进一步压缩该违约率的有效途径不是继续增大 $\chi$，而是改进累计能耗预测与校准。以上发现为下一章的总体结论与工程建议提供了直接支撑。
